## Context
In a Multi-Agent System (MAS) or federated repository structure, version control must handle nested "satellite" nodes that may be forks of external projects. This skill provides the patterns for managing these nodes without polluting the main orchestrator's history.

## Capabilities
- **Monorepo/MAS Hybrid**: Managing nested forked repositories using directory-based separation or git submodules.
- **Upstream Sync**: Logic for pulling updates from original external forks into the local system.
- **Agentic Commits**: Standards for AI-generated commit messages that include Loop IDs and turn-specific metadata.
- **Federation Mirroring**: Ensuring that `EVID:FED` entries are updated when remote head positions change.

## Expert Patterns

### 1. Nested Fork Management
Satellite repositories reside in `external_repos/`. They should be treated as independent versioned entities.
- **Pattern**: `git clone <fork_url> external_repos/<node_name>`
- **Exclusion**: Ensure `external_repos/` is tracked in the main repo either via submodules or by ensuring the main `.gitignore` allows the directory but ignores internal `.git` history if simple directory nesting is used.

### 2. Semantic Commit Standards (Agentic)
Every commit generated by an AntiGravity agent must follow this format:
```text
[<LOOP_ID>] <type>(<scope>): <short description>

- Rationale: <why this change was made>
- Evidence: <link to EVIDENCE file or hash>
- Turn: <turn index>
```
**Types**: `feat`, `fix`, `docs`, `skill`, `audit`, `harmonize`.

### 3. Upstream Handshake
When a forked repo has new changes:
1.  **Stash** local modifications (SAMPLES/SKILLSETS).
2.  **Fetch** from upstream origin.
3.  **Merge** or **Rebase** carefully, prioritizing local system protocols (`.ai/` directory takes precedence).
4.  **Verify** structural integrity using `ag_skillset_gen.py --dry-run`.
